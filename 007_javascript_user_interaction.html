Introduction to Javascript

Once a webpage has been created using HTML and CSS, its appearance in the screen will usually not change: the text in is text elements (paragraphs, headers, lists) will remain as is; the color of the text and backgrounds will too; same for images, video, forms, tables; the CSS grid that we designed to put the elements in, will remain unaltered as well.

In some situations, modifying the content of the webpage is desirable: for example, we may want to connect periodically to some other Internet webpage (e.g., our favorite news site), and grab some information from it (e.g., the breaking news headlines) and print that information in some area of our webpage. Or, we may want to allow the user to interact with the webpage in some way: for example, using a button to allow the user to change the background color of the webpage, or to add a paragraph with some information that the user has just written in some textfield (like a post in Twitter), etc. Javascript is a language that is used for these tasks involved in changing something in the webpage. These are sometimes called dynamic effects because they allow to change the content and presentation of a webpage according to the logic that the programmer wants. Webpages that use Javascript are thus sometimes called dynamic webpages. In contrast, webpages that use just HTML and CSS are called static webpages.

Javascript instructions are usually stored in text files with extension js. For example, a file named example.js would be a Javascript file. When we create an HTML file whose contents we want to control via Javascript, we need to tell the browser the path to the Javascript file that we want to use. We do this by means of the script element. Let's see it with an example: take a look at the following webpage, written in a file named exampleForAzu.html. This file defines four buttons, one that should change the webpage background color to red when clicked on, another to change it to green, another to blue, and another to white: 

<!doctype html>
<html>
  <head>
    <title>Azu is learning Javascript!</title>
    <script src="javascript4azu.js" defer></script>
    <link rel="stylesheet" href="css4azu.css">
  </head>
  <body>
    <input type='button' id="buttonForRed" value="Click here to change background to red">
    <input type='button' id="buttonForGreen" value="Click here to change background to green">
    <input type='button' id="buttonForBlue" value="Click here to change background to blue">
    <input type='button' id="buttonForWhite" value="Click here to change background to white"> 
  </body>
</html>

In the above HTML file, we declare the doctype (which indicates that we intend to use standard HTML), then the html element that contains the whole webpage, and inside it, two elements: the element head, and the element body. The element head, as we saw in previous lessons, contains information that is not directly shown to the user: in this example they are the title, a script element that indicates that we want to use a file named javascript4zu.js with Javascript code that we will use to make sure that the buttons allow to change the background color; and a link element referencing a CSS file named css4zu.css that would contain style instructions. Then, the element body contains everything that will be shown to the user in the webpage: in this example, it is just four buttons. We gave a unique ID to each one: the first button has an id of buttonForRed, and on it we write the phrase "Click here to change background to red"; when the user clicks on it, we want the background color of the entire webpage to turn to red; we will do that using code that we will write inside the Javascript file. The other three buttons are similar: one uses green, another blue, and the last one, white, which is usually the default color of any webpage background.

One important aspect of the script element is that it has a closing tag, and that nothing is written between the opening tag and the closing tag. This is in contrast to the link element that we use to indicate a CSS file, which does not have a closing tag.

The script opening tag has the optional attribute defer, which means that we want the browser to finish downloading the entire html file before attempting to process the Javascript file (i.e., the browser defers the processing of the javascript file until the html file has been downloaded). In general, it is recommended to write it, due to the following: the browser starts reading the html file at the beginning of the file, and processes it line by line. Take a look at the example: first, the browser processes the doctype, and then the opening tags of the html and head elements. Then, the title element, and then the script element referencing the Javascript file. Then, before continuing processing the link element that appears in the next line, it reads the entire Javascript file, and only when it finishes processing the Javascript file it will continue processing the html file, starting with the link element, then the closing head tag, and then the body element. The important part to notice here is that when the browser reads the Javascript file, it still did not read the four buttons that the Javascript file will manage (and that is because they are written in the html file AFTER the Javascript file). And, since the whole point of the Javascript file is to control those four buttons, the browser may get confused if we attempt to do anything related to those buttons inside the Javascript file. To make sure that the browser knows about the existence of the buttons when we try to control them with Javascript, it is better to defer the reading of the contents of the Javascript file until the entire HTML file has been processed (so the browser has alredy seen the buttons and everything else inside the webpage). That is why we write defer in the script opening tag.

Javascript Vocabulary

Now let's see how to write the Javascript file example4azu.js. Let's start with some Javascript vocabulary and grammar.

  1. The word document in lowercase means the entire webpage. NOTE: Javascript, in contrast to HTML and CSS, is case sensitive: the words document, Document, dOcUmEnT, etc., are all different.
  2. A dot between two words without spaces ---like in totoro.tail---- means that whatever word is on the right of the dot is applied to whatever is on the left. In the example before, that means the tail of the entity called totoro. You can think of the dot between two words as an apostrophe s in English.
  3. The expression document.title can be read as "document's title", and it means the title of the webpage.
  4. The expression document.body means all the HTML elements that we write inside the body tag in the HTML file.
  5. The word getElementById followed by a string between parentheses, like in getElementById("buttonForRed") means the HTML element whose id is the word in the parentheses. In the example HTML file above (the one called exampleForAzu.html) it would be the button whose id is buttonForRed.
  6. The word style means the set of CSS styling attributes of an HTML element, like color, background color, padding, etc.
  7. The word backgroundColor means the background color of some element. It is usually used jointly with the style keyword, like in style.backgroundColor.
  8. The equal sign is used to assign something a value. For example, the following would change the background color of the HTML element called "buttonForRed" to pink: 
        document.getElementById("buttonForRed").style.backgroundColor = "pink"
    The above is read by the browser as: "get the document, and inside it the element whose id is buttonForRed, and from its style properties, change the background color to pink".
  9. In Javascript, any word or series of words that isntruct the browser to do something is called an statement. For example, document.getElementById("buttonForRed").style.backgroundColor = "pink" is an statement because it tells the browser to do something. Is is customary, although not mandatory, to write a semicolon after each statement in Javascript, For example, instead of 
         document.getElementById("buttonForRed").style.backgroundColor = "pink"
    it is recommended to write 
        document.getElementById("buttonForRed").style.backgroundColor = "pink";
  10. In Javascript, any word or words that have any value is called an expression. For example, the word document is an expression; the series of words document.getElementById("buttonForRed").style.backgroundColor") is also an expression; the word "pink" is also an expression too.
  11. Javascript expressions are somewhat equivalent to anything that can be the subject in a normal language like English or Spanish. Javascript statements are equivalent to complete sentences with a verb that indicates an action.
    For example, document.getElementById("buttonForRed").style.backgroundColor") means "the background color of the element named buttonForRed", which can be the subject of an English sentence. The word "pink" can also be the subject of an English sentence. The complete expression 
          document.getElementById("buttonForRed").style.backgroundColor = "pink";
    is equivalent to a complete English sentence because it involves an assignment, which means "CHANGE to pink the background color of the element whose id is buttonForRed".
  12. The word innerHTML means whatever is written in between the opening and closing tags of a given HTML element. For example, if we have a paragraph written like this: <p>Hola caracola</p>, then its innerHTML woul be Hola caracola. This is, everything written between the opening and closing p tags will be its innerHTML.
  13. To append some text to a HTML text element, like a paragrap, or a list item, Javascript uses a plus sign followed, without spaces, by an equal sign, followed by the text to be appended. For example, the following code appends a phrase to the element whose id is parrafoSobreBurgos: document.getElementById("parrafoSobreBurgos").innerHTML += " El cuento de la buena pipa."
  14. Javascript can be used to tell the browser to do some action when a condition of some kind occurs. For example, imagine that we want to do the following: when the user clicks the button whose id is "curiousButton", then the browser should change the color of the background of the webpage to blue, and the color of the font of the text of all elements to red, and append the phrase "Burgos has a town." to the paragraph whose id is "parrafocastilla", and replace the text of the paragaph whose id is "parrafogalicia" for the phrase "Galicia has four provinces. In Javascript, any group of actions that we want to do one after the other is called a function. The Javascript programmer is free to choose a name for any function. A function is indicated by the word function, followed by the chosen name for the function, two parentheses, and the actions to be done written inside braces. It is usual to write one action per line. In the example above, if we choose the name curiousFunction for the actions that we described above, we would write: 
        function curiousFunction() {
      document.body.style.backgroundColor = blue;
      document.body.style.color = red;
      document.getElementById("parrafocastilla").innerHTML += "Burgos has a town.";
      document.getElementById("parrafogalicia").innerHTML = "Galicia has four provinces.";
    }
    IMPORTANT: observe the difference between how we use += and =: the former appends text to the existing text, and the latter replaces the existing text for new one. Take care with not confusing them.
  15. When the browser sees a function like in the previous point (this is, declared using the word function) it does NOT do the actions indicated; it just tells the browser what actions we want to do in the future when we desire so. So, when the browser sees the the previous example, the background color would not change, nor would the text color, etc. In order for those actions to happen, we need to write the name of the function followed by parentheses, like this: 
           curiousFunction();
  16. In Javascript, the word onclick means the actions that happen when the user clicks on a given element. For example, if we write the following inside a Javascript file: 
          document.getElementById("curiousButton").onclick = curiousFunction;
    it means that we are telling the browser that, when the user clicks on the webpage's element whose id is curiousButton, it should do the actions written inside the function called curiousFunction. When the browser reads the previous line, it does not do those actions: it will only do them when the user clicks on the button. It is important to note that there are not parentheses after the name curiousFunction; if there were, the actions would be done when the browser reads that line (but it would be incorrect, so do not put parentheses there).
  17. In Javascript, three equal signs one after the other without blank spaces in between them, mean the question is?, For example, the expression document.title === "Galicia" means is the document's title Galicia?.
  18. If you want to do something only if something else is true, you can use what is called an if statement. For example, if you want to change the background color of the webpage to blue ONLY if the current webpage title is Galicia, you can write the following: 
          if (document.title === "Galicia") {
          document.body.style.backgroundColor = "blue";
      }
  19. If you want to do something ONLY if something else is true, and in addition you also want to do a third thing if that other thing is false, you can use what is called an if/else statement. For example, if you want to change the backlground color to blue if the title is Galicia, and change it to green if the title is not Galicia, you can write: 
          if (document.title === "Galicia") {
          document.body.style.backgroundColor = "blue";
      } else {
          document.body.style.backgroundColor = "green";
      }
  20. If you want to do different things depending on the value of something, you can use what is called an if/else if statement. For example, if you want to change the background color to blue if the title is Galicia, and change it to green if the title is Asturias, and to red if it is Castilla, and to magenta if the title is Cantabria, and to white in any other case, you can write: 
          if (document.title === "Galicia") {
          document.body.style.backgroundColor = "blue";
      } else if (document.title === "Asturias") {
          document.body.style.backgroundColor = "green";
      } else if (document.title === "Castilla") {
          document.body.style.backgroundColor = "red";
      } else if (document.title === "Cantabria") {
          document.body.style.backgroundColor = "magenta";
      } else {
          document.body.style.backgroundColor = "white";
      }
    You can write as many } else if ... lines as needed. The previous code can be read as: if the webpage's title is Galicia, change its background color to blue; if it is Asturias, to green, if it is Castilla, to red, if it is Cantabria, to magenta, and if it was not any of those cases, to white.
  21. In a Javascript file, anything is written between the strings /* and */, is ignored by the browser. That is called a comment, and it is usually used to give information for the human reader of the file: author information, copyright notice, an observation that may be useful for the future reader of the file, etc. For example: 
         /*Hello everybody: 
       this file was created by the cookie monster
       Copyright hold by Tedio Plomez Sopor*/
     document.body.style.backgroundColor = "magenta"; /* we choose magenta because we like it*/
  22. In Javascript, you have access to space to store pieces of data that you may want to use inside a webpage. You can assign any name to any piece of data, and give it whatever value you want. You can also reassign the values as many times as you want. Thus, these pieces of data are called variables. For example, if your webpage wants to show information about a city, you can have a variable named city_name, whose value is the name Tokyo, and another variable named part_of_the_world, whose value is the phrase East Asia, and another variable named population, whose value is the number 40000000, and another variable named lottery_number, whose value is the set of characters 40000000, which means 40 million, but we do not want to trear as a number, but rather as a name (so no mathematical operations on it), and another variable named famous_neighborhoods, whose value is the set of values Minato, Chuo, Shinagawa, Shibuya, Shinjuku, and another variable named is_country_capital, whose value is the truth value true. Names are written between quotes, numbers, if we want to do nathematical operations with them later, are written without quotes, and with quotes if we do not want so. Sets of values are also called arrays, and are written between squate brackets, with a comma separating each value. Truth values are called booleans (in honor of James Boole, an Irish logician) and can be either true or false. Each piece of data is written using the word let followed by the name, an equal sign, and the value, like this: 
          let city_name = "Tokyo"; /* This variable value is a name, so it uses quotes*/
      let par_of_the_world = "East Asia";  /* This value is a phrase, so it uses quotes */
      let population = 40000000; /* This is a number that admits mathematical operations, so it does not use quotes*/
      let lottery_number = "40000000"; /* This is a number that does not admit mathematical operations, so it uses quotes*/
      let famous_neighborhoods = [ "Minato", "Chuo", "Shinagawa", "Shibuya", "Shinjuku" ]; /* this is an array*/
      let is_country_capital = true;
    Variable names can contain letters, numbers, and underscores. The first character must not be a number. It is possible to change the value of any of these pieces of data at any point inside the Javascript file, for example: 
          ...
      population += 10000000; /*population changes from 40 million to 50 million*/
      famous_neighborhoods += [ "Akasaka" ]; /*the old array, which had 5 elements, is extended with a new value, so it now has 6 elements*/
  23. Javascript offers a tool called loops to control how many times some action is done. One of the most useful loops is called the for loop. Let's see it with an example. Let's assume that there is inside the HTML file an element whose id is paragraph_about_cats. In it, we want to write 100 phrases, in the following style: 
          There are 1 cats in the bed.
      There are 2 cats in the bed.
      There are 3 cats in the bed.
      ....
      There are 98 cats in the bed.
      There are 99 cats in the bed.
      There are 100 cats in the bed.
    We can write the above directly inside the HTML file, but is is going to take a lot of work (100 lines to be written). Instead, we can see that there is an easy pattern in those phrases: all phrases have exactly the same format, and only the number changes in each one, from 1 to 100. This is a clear pattern; and for loops are designed to create patterns. In this case, we can werite inside the Javascript file the following: 
        for (let number_cats = 1; number_cats <= 100; number_cats = number_cats + 1) {
       document.getElementById("paragraph_about_cats").innerHTML += "There are " + number_cats + "cats in the bed.";
    }
    The for loop has the word for followed by, between parentheses, three statements separated by semicolons, and one or more statements between braces. The first statement inside the parentheses (in this example, let number_cats = 1;) declares a variable that we named number_cats; this first statement in called the initialization statement of the for loop: the browser runs the initialization statement once and only once, at the beginning of the processing of the for loop; in this case, the variable number_cats is assigned a numeric value of one. The for loop has a second statement, in this case, number_cats <= 100;. This is called the termination statement: the browser, after running whatever we wrote between the braces, checks if the termination statement is correct; in this case, whether the variable number_cats is lower than or equal to 100. If that is false, it stops processing the for loop; if it is true, it continues. The third statement inside the parentheses is called the increment statement; in this case, it is number_cats = number_cats + 1. The browser runs the increment statement after finishing doing all the actions written between the braces. So, in this case, we can translate to English the above example like: initialize a variable called number_cats to the number 1, and then append to the HTML element called paragraph_about_cats the phrase There are followed by the value of the variable number_cats (which is 1), followed by the phrase cats in the bed.; then increment number_cats by one (so it becomes the number 2); then check whether number_cats is less than or equal to 100; if that is false; finish the for loop; else, append, to the HTML element called paragraph_about_cats the phrase There are followed by the value of the variable number_cats (which is 2), followed by the phrase cats in the bed.; then increment number_cats by one (so it becomes 3); then check whether number_cats is less than or equal to 100; if that is false; finish the for loop; else, append, to the HTML element called paragraph_about_cats the phrase There are followed by the value of the variable number_cats (which is 3), etc. We can see that this will append the phrases "There are N cats in the bed." until N is incremented from 100 to 101, at which point the check for whether number_cats is less than or equal to 100 will fail, so the browser will stop the for loop and stop adding phrases with numbers.
  24. Javascrip also has another kind of loop called the while loop. It is equivalent to the for loop, but not as expressive, For example, to do the previous example with a while loop we would write: 
        let number_cats = 1;
    while (number_cats <= 100) {
           document.getElementById("paragraph_about_cats").innerHTML += "There are " + number_cats + "cats in the bed.";
           number_cats = number_cats + 1;
    }
    The previous can be read in English as: create a variable named number_cats and set it to the number 1; then, while the value of number_cats is lower than or equal to 100, append to the document the phrase There are plus the value of the variable number_cats plus the phrase cats in the bed. and then increase the value of the variable number_cats by one.
  25. Numeric variables can be incremented in different ways: 
       let i = 100; /* This declares a variable named i, whose value is 100 */
   i = i + 1; /* This adds 1 to the value of i and stores the value in i; so i becomes 101 */
   i++; /* This is equivalent to the above line, i.e., shorthand for i = i+1; so i becomes 102*/
   i = i + 5; /* This adds 5 to i, so i becomes 107 */
   i += 10; /* This adds 10 to i, so i becomes 117*/
   i = i - 100; /* This subtracts 100 from i, so i becomes 17*/
   i -= 7; /* This subtracts 7 from i, so i becomes 10 */
   i = i * 2 ; /* This multiplies io by 2 and stores the value in i,so i becomes 20 */
   i *= 3; /* This multiples i by 3, so i becomes 60*/
   i = i / 3; /* This divides i by 3, so i becomes 20 */
   i /= 5; /* This divides i by 5, so i becomes 4*/
   
   let j = 2; /* This declares a variable named j with a value of 2*/
   let k = j * i; /* This declares a variable named k whose value is the multiplication of the values of the variables i and j, so k becomes 8*/
  26. There is a loop called do while loop that is written 
      let number_cats = 1;
  do {
           document.getElementById("paragraph_about_cats").innerHTML += "There are " + number_cats + "cats in the bed.";
           number_cats = number_cats + 1;
  } while (number_cats <= 100);
    This is read: declare a variable named number_cats and initialize its value to 1, and then, do whatever is in the braces, and then check if number_cats is lower than or equal to 100; if it is so, do the actions between brackets again, othrwise, stop the loop. The difference of this do while loop with the while loop is that the former always does the actions between brackets at least once (because the termination check is written after the brackets) while the latter may not run the actions between the brackets if the termination check fails (because it is written before the brackets).

  } else if (document.getElementById("thirdparagraph").style.color === "yellow") {
     document.getElementById("thirdparagraph").style.color = "pink";            
  } else if (document.getElementById("thirdparagraph").style.color === "pink") {
     document.getElementById("thirdparagraph").style.color = "green";           
  } else {                                                                      
     document.getElementById("thirdparagraph").style.color = "black";           
  }                                                                             
                                                                                
  setTimeout(change_style_of_third_paragraph_each_second, 1000);                
}
