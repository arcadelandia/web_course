HTML (acronym of hypertext markup language) is a way to tell a web browser two things:

  1. what information we want to show in a webpage, and
  2. how to represent that information on the screen.
With HTML, we tell the browser the text that we want in the webpage, plus any links to other pages, plus images, plus videos, plus buttons, plus forms to allow the viewers of the webpage to input information, etc. These elements are collectively called hypertext.

HTML does not care about the amount of whitespace between words or about how many lines we have in a document. If we write a file with very long lines, and another with the same content but with very short lines, and another with several whitespaces between words, and another with just one whitespace between words, and finally we open them with a web browser, the four files will be represented in the screen as exactly equal to each other. If we want to control the presentation of the text (line length, separation between lines, centering of text, font color, etc.), we will need to use another language called CSS (we will learn it in a later lesson).

HTML is what is called a markup language, because it uses special words or sequences of characters (called markup in the world of text edition) to instruct the browser about how it should present the webpage to the viewer. Each markup language uses its special words. HTML does too; for example, the markup word b means to bring the attention of the webpage reader to the text that follows (browsers usually interpret that by writing the text in bold); in a related example, the markup word /b means to stop bringing the attention of the reader to that part of the text (so, if the browser was writing text in bold until that moment, it stops doing so). In HTML, the b is called an opening markup word, and the /b is called a closing markup word. The difference between an opening and a closing markup word is the upwards slash at the beginning of the closing markup word.

In HTML, to tell the browser that we want to use a markup word, we enclose the word inside the "less than" and "more than" symbols. For example, to use the markup word b, we would write it as: <b>. This is needed because otherwise it would be impossible to write the letter b as a separate word by itself in a document, as the browser would interpret it as a command rather than a a word to be shown on the screen. For example, if we want to write the list of the four seasons, with the middle two written in bold font, we would create an HTML document with the content: spring <b> summer autumn </b> winter. The browser will write in the screen the names of the four seasons, bolding the two middle words, and nothing else: in particular, it will not show in the screen neither <b> nor </b>.

In another example, the markup word em means to emphasize the content that follows (browsers usually do this by using italics); the markup word /em means to stop emphasizing the text. Markup words can be combined; for example, if we want to write a phrase in bold and italics at the same time, we would use the b em markup words together (in any order we want), write the text that we want to write, and then use /b /em to tell the browser to stop using bold italics. When using several markup words together, it is usually recommended to write the closing markup words in reverse order to the opening one. For example, both the phrase spring <em> <b> summer autumn </em> </b> winter and the phrase spring <em> <b> summer autumn </b> </em> winter are correct, but the second is usually preferred when creating HTML because the closing tags are in reverse order to the opening ones (this is, we opened first with em and second with b, and closed first with b and second with em).

An HTML document is usually saved in a plain-text file whose name ends with either .html or .htm, although .html is preferred among those two extensions. If we write any other extension at the end, and we open that file with the web browser, the web browser will not interpret its contents as markup language, and will show instead the plain-text of the page. For example, if we write "One <b> two </b>" in a file called example.html and open it in a web browser, the web browser will show the word One without bolding it, and the word two in bold; however, if we call the file example.txt and open it with the browser, the browser will not pay attention to the markup words, and instead it will show everything in the screen, both markup words and normal words.

HTML calls elements to its markup words. Programmers often also call them tags. Some of the most common HTML tags are:

  1. title: instructs the browser which title to show on the tab where the webpage is opened (usually above the page and menus).
  2. p: instructs the browser to create a new paragraph (the browser will usually put more vertical whitespace than inside a praragraph)
  3. h1: instructs the browser to create a header
  4. h2: instructs the browser to create a slightly smaller header than with h1.
  5. h3: instructs the browser to create a slightly smaller header than with h2.
  6. br: (break) instructs the browser to force a line break (usually to write poetry, or anything that requires a line break at a given point). It does not have a closing markup tag, as it does not make sense.
  7. b: instructs the browser to bring attention to the text starting at that point (usually means bold, unless in reading devices for the blind, etc)
  8. em: instructs the browser to emphasize the text starting at that point(usually means italics).
  9. u: instructs the browser to underline the text that follows.
  10. ol: (ordered list) instructs the browser to create a bulleted ordered list (where each item is preceded by a number or letter).
  11. ul: (unordered list) instructs the browser to create a bulleted unordered list (where each item is preceded by an asterisk or a dot).
  12. li: (list item) instructs the browser to create a new item for the ordered or unordered list where it is located.

Most markup words need opening and closing version (e.g., b and /b, p and /p, or h1 and /h1), because we need to tell the browser when to start and end doing what we want it to do. For example, when bolding text, we need to tell when to start, and when to stop; idem for writing a paragraph, etc. However, some markup words do not need a closing markup tag because it does not make sense: for example, br inserts a new line, so it appears by itself whenever when we want to insert one.

Example HTML code

Below is an example webpage using some of the markup elements we described above. 

<title>Continents of the world</title>                              
<h1>The continents and trivia about them</h1>                       
<p>The world has six continents</p>                                 
<ul>                                                                      
  <li> Europe </li>                                                 
  <li> Asia </li>                                                   
  <li> Africa </li>                                                 
  <li> America (sometimes separated into North America and South America)</li>
  <li> Oceania </li>                                                
  <li> Antarctica </li>                                             
</ul>                                                                     
<h1>Trivia about the continents</h1>                                
<p> Europe has many countries, among them Spain, Portugal, and France. </p>
<p> America is sometimes considered formed by two separate continents: North America and South America.</p>
<p> Asia is the most popuolous continent. Both India and China have each more than <b>1 billion</b> people.
  Asia has the Himalayas mountain range, which contains the highest mountains of the world. By the way,
  the three highest mountains of the world are: <ol> <li> Everest </li> <li> K2 </li> <li> Kangchenjunga </li> </ol>
</p>                                                                      
<p>Africa borders the Mediterranean Sea. It has the Sahara, and large rainforests and savannahs.</p>
<p>Oceania has a lot of small islands, and also Australia, the biggest island in the world.</p>
<p>Antarctica has a huge ice cover; it hosts the South Pole, and also has plenty of penguins.</p>
<h1>More trivia about Europe</h1>                                   
<p>The following is a list of trivia per country of Europe.</p>      
<h2>Spain</h2>                                                      
<p>Spain has 17 autonomous communities.</p>                         
<h2>Portugal</h2>                                                   
<p>Portugal uses Portuguese as official language.</p>               
<h3>North regions of Portugal</h3>                                  
<p>The north of Portugal limits Galicia.</p>                        
<h3>South regions of Portugal</h3>                                  
<p>The south of Portugal does not limit Galicia.</p>                
<h2>France</h2>                                                     
<p>Paris is the capital of France.</p>                              
<p>Grenoble is near the Alps. It is close to Mont Blanc, which  is the highest mountain in the Alps.</p>
<h2>Germany</h2>                                                    
<p>Germany borders the North Sea and the Baltic Sea.</p>            
<p>The North Sea is part of the Atlantic Ocean, like the Cantabric Sea.</p>
<p>The Baltic Sea is cold.</p>

EXERCISES

    Steps of the exercise:
  1. Create a new folder to contain all the files that we will create in these exercises.
  2. Inside the folder created above, create a new file in notepad++ named 001_continents.html, and write into it the example HTML code we shown above.
  3. Read it, observe the usage of all the elements, and try to predict how they will be represented in the screen once the browser loads that file.
  4. Open the file you just created in the browser (usually by typing Ctr+O, or using the "Open File" option in the menu of Chrome or any other browser). Observe how each individual markup element of the HTML file gets represented in the screen. Pay attention to the title,h1,p,ul,li,h2,h3 tags in all the places they appear.
  5. Copy, with the file explorer, the file 001_continents.html to another file named 001_continents.txt, so their contents are the same and their names only differ in the extension. Open this 001_continents.txt file in another tab inside the browser. Can you see that the browser is just showing the text you wrote? It is not interpreting the markup tags as commands; instead, it just represents in the screen what was written. That is because the extension is "txt" rather than "html".
  6. Using notepad++, modify the first line of the 001_continents.html file, changing the closing markup tag </title> for <title> (so it is the same as the opening markup tag), save the file, and reload it in the browser (or open it in a tab if you closed it before). You should see that the whole page is blank. If you pass the mouse through the tab title, you should see a snippet of text with the entire HTML code there (which is the new title of the webpage). What happened is that the browser started readng the file at the very beginning, where the first <title> tag appears, analyzed it and correctly interpreted it as the start of the title that we want to use for the webpage. From that point on, the browser treats everything that appears before the closing markup tag </title> as the webpage title. However, there is no closing markup tag </title> because you changed it, so the browser continues reading everything until the end of the document and interprets it as the title, and shows it in the tab, above the webpage. Then, as the browser has reached the end of the HTML file, and there is nothing else to read, the webpage that we see is entirely blank. This is not what the page programmer wants to do; in general, any undesired behavior in programming is called a bug. The process of analyzing what is wrong when we find a bug, is called debugging. In this case, the bug was caused by an incorrect closing markup tag for the title tag. Thus, when programming in HTML, it is very important to pay attention to the correct typing of all the opening and ending markup tags. As this can get tedious, especially when creating large HTML files, there are tools designed to analyze the HTML documents that you write and that tell you if you forgot something. We will use them in other lessons.
  7. Edit back the first line, changing the closing markup to </title>, as it was originally, and reload it again in the browser, so you see it normally again.
  8. For each day of the week, create an example HTML webpage that covers some of the above elements, about any theme you want, and save it with descriptive file names in the same folder where you have the 001_continents.html file (for example, 001_music.html, 001_geography.html, 001_blablabla.html, etc.).
  9. End of the exercises.
