We are going to learn how to use CSS, a progamming language that controls how the elements of a HTML webpage are presented.

CSS is a programming language that allows to tell the browser how to change its default behavior when presenting HTML webpages to users. Its name is an acronym for "cascading style sheets": an style sheet is a set of attribute names plus its corresponging values (font color, font size, margin size, etc). For example, a style sheet can say that paragraphs must be written in a blue font over a yellow background. Cascading means that a HTML page can use several style sheets at the same time, and any potential conflicting instructions can be resolved in a reasonable manner: for example, if one style sheet says that font color should be blue, and another that it should be red, the browser has some reasonable rule to determine which one, if any, should be the chosen one.

HTML, as we have seen until now, allows us to define some elements by means of tags: for example, we use a <p> to indicate that we want to write a paragraph, a <h1> for a heading, a <ul> for an unordered list, etc. Those tags tell the browser the kind of information that we want to provide: when we use <p>, the browser will automatically put more space with respect to whatever we were writing before; when we use <ul> the browser prepares to print a list of items in vertical; when we use <h1> the browser uses a large font and usually writes that headline in bold font, etc.

By default, all the elements that we write in the HTML file are presented in the screen in the same order in which we wrote them in the HTML file.

CSS allows to present elements in the browser in an order that is not the one present in the HTML file: for example, if we write in HTML an ordered list of links to pages, the browser will write that list immediately below what was written before it in the HTML file. However, with CSS we can tell the browser to put it somewhere else: for example, to put it in the quarter left side of the window, preventing any oter content to share that space: this would turn that ordered list into a navigation menu of the kind that is common in many websites. We can also use CSS to put it in the right of the page, or on the top, or on the bottom, etc.

We can also use CSS to indicate the size of margins of elements, whether we want borders around them and with which styles (dashed, solid, dotted, etc.), font colors, font sizes, background colors, font families and styles (underlines, overlines, with a line through the text, cursive, bold, bold size, etc.).

CSS also allows control over the shape and color of the mouse cursor when it hovers over a given element (for example, change the cursor from an arrow to a dot when hovering over a given image, etc.).

There are too many CSS properties to cite them here. For a good introduction to them, search Google for "CSS MDN" (MDN stands for Mozilla Development Network; Mozilla, a play on Godzilla, is an organization that has been involved in the development of web technology since the 1990s).

CSS instructions are written in one or more files. Those files are usually named with ".css" extension. To tell the browser to apply the instructions found in a file named "aeiou.css" file to a given HTML file, we write anywhere in that HTML file the following: 

<link rel="stylesheet" href="aeiou.css">
This basically means that the HTML webpage has a relationship to a file whose type is a stylesheet (the name for CSS technology) and the name of the file is "aeiou.css".

Inside the css file, we write instructions that will tell the browser how to modify the appearance and presentation in the screen of the content of the HTML webpage. There are three main ways to do that inside the CSS file:

  • If we want to control all elements with a given tag name (for example, we we want all paragraphs to use a blue font and a red background) we write: tagname {property1: value1; property2: value2; ... ; propertyN: valueN;} where tagname is the name of the tag whose appearance we want to control, propertyX is the name of the property that we want to control, and valueX the value for that property. For example, to tell the browser that elements with tag p should have blue font over yellow background, we would write: 
    p {
   color: blue; 
   background-color: yellow;
}
  • If we want to apply some instructions to a specific element instead of to all, we need to identify it by an unique name inside the HTML code, and then refer to that unique identifier in the CSS file. In order to give a unique name to a given element in the HTML code, we use the attribute "id" inside the opening tag of that element. For example, the following: 
       <p id="spain">Spain is in Europe. It borders the Atlantic Ocean.</p>
    defines a paragraph with the identier "spain". Idenfiers are case sensitive, and must be unique inside the HTML document: no other element can have the "spain" identifier. Now we can write this in the CSS file: 
       p {
      color: blue; 
      background-color: yellow;
   }
   #spain {
      color: yellow; 
      background-color: red;
      text-decoration: underline;
   }
    And in this way, the paragraph identified as "spain" would have yellow font over a red background, and its text would be underlined. As the CSS file also indicates that paragraphs should be written in blue font over yellow background, there is a potential conflict between the instructions. In this case, the browser uses the ones for the "#spain" identifier because it is more specific than the ones for "p".
  • If we want a lot of elements in a HTML page, but not all, to have a common appearance, in theory we can put a different id to each of them, and then indicate, for each one, how we want them to appear. For example, if we have a HTML page where we have a paragraph for each country in the world, and we want to use a common color for the countries of each continent, then we would have 193 paragraphs: 43 would be for countries in Europe, 35 for America, 54 for Africa, 47 for Asia, and 14 for Oceania. We can give a unique id for each paragraph in the HTML file (whose value can be, for example, the name of that country), by doing something like this: 
         <p id="spain">Spain is square.</p>
     <p id="portugal">Portugal is rectangular.</p>
     ...
     <p id="japan">Japan is long.</p>
     <p id="china">China is rectangular.</p>
     ...
     <p id="morocco">Morocco has the Atlas.</p>
     <p id="algeria">Algeria is big.</p>
     ...
     <p id="canada">Canada is cold.</p>
     <p id="usa">USA is rectangular.</p>
     ...
     <p id="austalia">Australia is big.</p>
     <p id="newzealand">New Zealand has two islands.</p>
    and then, in the CSS file, we could have 193 repetitions of something like this: 
          #spain {color: blue; background-color: white; font: 2em; font-family: arial; }
      #portugal {color: blue; background-color: white; font: 2em; font-family: arial; }
      ...      
      #japan {color: red; background-color: green; font: 2em; font-family: arial; }
      #china {color: red; background-color: green; font: 2em; font-family: arial; }
      ....
      #morocco {color: blue; background-color: green; font: 2em; font-family: arial; }
      #algeria {color: blue; background-color: green; font: 2em; font-family: arial; }
      ....
      #canada {color: red; background-color: blue; font: 2em; font-family: arial; }
      #usa {color: red; background-color: blue; font: 2em; font-family: arial; }
      ....
      #australia {color: yellow; background-color: white; font: 2em; font-family: arial; }
      #newzealand {color: yellow; background-color: white; font: 2em; font-family: arial; }
    We can see that all countries of the same continent share the same presentation attributes. That is a totally fine way to control the appearance of the HTML page. However, in order to save typing in the CSS file, we can instead group all HTML elements of a given continent in what is called a HTML class, and use CSS to tell the browser to apply the same instructions to all elements that belong to a given class. We use the attribute

    class

    in the opening tag of an HTML element to identify it as belonging to a class. In our example, we could create five classes, one per continent, and apply them to each paragraph in the folowing way: 
         <p class="europe">Spain is square.</p>
     <p class="europe">Portugal is rectangular.</p>
     ...
     <p class="asia">Japan is long.</p>
     <p class="asia">China is rectangular.</p>
     ...
     <p class="africa">Morocco has the Atlas.</p>
     <p class="africa">Algeria is big.</p>
     ...
     <p class="america">Canada is cold.</p>
     <p class="america">USA is rectangular.</p>
     ...
     <p class="oceania">Australia is big.</p>
     <p class="oceania">New Zealand has two islands.</p>
    And then, instead of having 193 lines in our CSS file, we can have just 5, where each line tells CSS how to represent each element of a class. To do that, we write a dot followed by the class name: 
          .europe {color: blue; background-color: white; font: 2em; font-family: arial; }
      .asia {color: red; background-color: green; font: 2em; font-family: arial; }
      .africa {color: blue; background-color: green; font: 2em; font-family: arial; }
      .america {color: red; background-color: blue; font: 2em; font-family: arial; }
      .oceania {color: yellow; background-color: white; font: 2em; font-family: arial; }
    It is possible to indicate both class and id for the same element; if we want a specific element to have a differnet presentation than the rest of the elements of its class, then we need to give it an id, For example, if we wanted to write the paragraph for australia in the same style as before but with yellow font over green background, we would write: 
         <p class="europe">Spain is square.</p>
     <p class="europe">Portugal is rectangular.</p>
     ...
     <p class="asia">Japan is long.</p>
     <p class="asia">China is rectangular.</p>
     ...
     <p class="africa">Morocco has the Atlas.</p>
     <p class="africa">Algeria is big.</p>
     ...
     <p class="america">Canada is cold.</p>
     <p class="america">USA is rectangular.</p>
     ...
     <p id="australia" class="oceania">Australia is big.</p>
     <p class="oceania">New Zealand has two islands.</p>
    And now we would write in the CSS: 
          .europe {color: blue; background-color: white; font: 2em; font-family: arial; }
      .asia {color: red; background-color: green; font: 2em; font-family: arial; }
      .africa {color: blue; background-color: green; font: 2em; font-family: arial; }
      .america {color: red; background-color: blue; font: 2em; font-family: arial; }
      .oceania {color: yellow; background-color: white; font: 2em; font-family: arial; }
      #australia {color: yellow; background-color: green; font: 2em; font-family: arial;}
