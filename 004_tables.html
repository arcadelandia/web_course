In this lesson we will learn how to create HTML tables. Tables are a popular way to present information: they have a series of rows where each row is composed by one or more columns of information. An HTML table is created using the tag table. Tables usually have:

  • an optional caption that tells the reader what the table is about. It is created with the tag caption
  • an optional header row where each column has a label that allows the reader to know what that column is about. It is created with the tag thead
  • a body composed by all rows of information that the table will show. It is created with the tag tbody
  • an optional footer row where each column usually summarizes the information of that column. It is created with the tag tfoot

Each row of information in the header, body, and footer is created with the tag tr. Each column of information inside a row is created with either the label th (short for table header, which makes the information be written in bolder font) or with the label td (short for table data, which is the normal one), as decided by the programmer creating the HTML table.

Let's look at the following example table: 

<table>                                                                   
<caption>Population of the planets of the Solar System</caption>    
<thead>                                                                   
        <tr>
                <th>Name of the planet</th>                                 
                <th>Year 2021</th>                                          
                <th>Year 2022</th>                                          
                <th>The future</th>                                         
        </tr>
</thead>                                                                  
<tbody>                                                                   
        <tr>                                                              
                <td>Mercury</td>                                    
                <td>20</td>                                         
                <td>31</td>                                         
                <td>52</td>                                         
        </tr>                                                             
        <tr>                                                              
                <td>Venus</td>                                      
                <td>50</td>                                         
                <td>Muchos</td>                                     
                <td>Pocos</td>                                      
        </tr>                                                             
        <tr>                                                              
                <td>Earth</td>                                      
                <td>30</td>                                         
                <td>9 billion</td>                                  
                <td>Only fish</td>                                  
        </tr>                                                             
</tbody>                                                                  
<tfoot>                                                                   
        <tr>                                                              
                <th>Total:</th>                                     
                <th>80</th>                                         
                <th>A lot</th>                                      
                <th>A little</th>                                   
        </tr>                                                             
</tfoot>                                                                  
</table>

When writing a table, it is important to pay attention to pair the opening and closing tags correctly (for example, that each th is terminated by a /th, and that each td is terminated by a /td, and that each tr is terminated by a /tr), otherwise the browser may get confused and show data in incorrect places.

Tables may have lots of information; thus, when writing their HTML code, it is advisable to write it in a way that is easy to read. What is wasy to read is subjective, but many programmers tend to use short lines and putting several td elements one on top of each other instead of beside each other and using indentation for different elements; the example above is written that way. When creating an HTML page, you will have lots of options to decide how much to put in each line and how to indent it; try to experiment to see how it appears to be more readable.

EXERCISES:

  1. Read the above HTML code and try to figure out how it should be represented in the browser.
  2. Write the above HTML code in a file, and open it with the browser. Observe how the fields are displayed. The columns are probably too tightly spaced, and the separation of the caption with respect to the table is probably too short; in addition, no lines are usually present separating the header from the data rows, nor the data rows from the footer. All these presentation elements (lines, separations, whether text is centered in each column or aligned to one side, colors, etc.) will be controlled using the CSS language, which we will see in another lesson.
  3. Write a sample HTML page with a table that lists, for every UE country, its name, its capital, its population, and a popular dish.
